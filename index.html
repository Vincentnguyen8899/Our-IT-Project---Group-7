Project description:

Overview:

Topic:
Many people in Vietnam are currently interested in the topic of cuisine. The number of people watching food videos on popular social media platforms like YouTube and TikTok is rapidly increasing. Not only do Vietnamese people enjoy learning the recipes for delicious dishes from all over the world, but they also enjoy learning the recipes for dishes from all over the world.
As a result, our team decided to create a playground where foodies could interact and share recipes, as well as a massive culinary library entirely devoted to Vietnamese people. Learning and exchanging experiences, saying no to negativity, and being user-friendly are all criteria.

Motivation:
Vietnamese people who enjoy trying new dishes from all over the world will likely benefit from the project. For instance, our mother occasionally searches the internet for new cooking recipes, but they always question how to save or share them with neighbors and relatives. So we decided to create RecipeBook, which is designed to be simple to use by people of all ages. With some additional app functions, it allows people to save recipes they want to try cooking or easily post their recipes.

Landscape:
First of all, RecipeBook is designed for users, unlike most of other Cooking Recipe websites, 100% recipes here are uploaded by users. If other users love the recipe, that post will be more likely to be suggested in other user walls.
Second, the posting feature here is specialized and is very easy for users to use. It doesn’t just contain a page that users have to type and organize everything in order to post a beautiful recipe, it contains tabs, photos, suggestions that organizes everything so even an 8-year-old child can post a recipe (but if no one likes it, it won’t appear much in other users' walls).

Detailed Description:

1. Aims:

The aim of this project, as well as the first milestone of this app, is to implement a fully functional version on the website that contains features, which will be stated later in this report. The goal is to develop and test all the core features of the prototype as if it is ready to be published. In order to obtain this aim, sub-goals will be created to encourage the team members to work on their best.
Sub-goals:

  • Account: For this goal to be reached, testers must be able to register and log into the application. The account must be able to send and receive users’ information from the application’s database. The account feature must also be able to receive token from Facebook and automatically generate an account for uses.
  • Profile: This is a part of the information which will be received from the database. The main goal of this feature is to arrange and display different layers of information in a given preset. The information that will be received contains text (user’s name, bio, nickname, etc), date (date of birth, date joined) or image (avatar, background photo, etc).
  • Posting: This feature is basically a personal database of each account that stores data of user’s posts since the creation of that account. The data that will be stored in this database are Title (text), Ingredient (list of text), Unit (list of number representing ingredient amount) and Step (long text).
  • User interaction: The most challenging part of this project is to implement a fully functional and stable interacting system as it will directly impact the user’s experience. All the interactions must be executed correctly with low latency and proper return from the application.
  • Extra features: Along with the core functionality, we have decided that these functions will also be fully implemented in this project:
    • Search bar: Users should be able to use keywords such as the name of dishes, ingredients, or cooking method to find the desired posts. Filters can be added to optimize the user experience.
    • Notifications: This function sends push notifications through web browsers to notify users about their activities on the app (reactions, or new followers).
    • Built-in PDF converter for posts: a built-in converter that can convert a post into a printable PDF file that can be printed for offline storage or physical transportation.

2. Plans and Progress:

The original idea of this project, which came up by Chi Toan, was to create a platform for home cooks to share their recipes with others. The contents posted by users will have optimized arrangements which make it easy to write as well as easy to read. To bring this platform alive, we decided to divide the progress into different milestones to improve work efficiency. Due to the situation of the pandemic, the development of the platform is being slowed because of the difficulties in communication and interaction between team members.
The first phase is to create the web version of the apps with a functional UI and core databases. The main tasks of this phase are developing and having insiders test the system. At the end of this phase, the web version of the platform should be published and ready for phase two.
In phase two, also known as “The engagement phase”, when the platform is published. The marketing campaign will be implemented and user’s feedback will be collected for improvement. The goal of this phase is to earn a stable amount of users and income. These results will directly impact to the progress of phase three.
In phase three, which is also the final phase, the team will concentrate on creating the mobile app of the platform along with improving the current user’s experience. This will be the most challenging phase as the mobile version and the web version must be fully synchronized to keep the user experience fluent. From this phase, every member of the team will focus on improving the app as well as adding new features.

4. Scope and Limits:

The project will cover the following features of the application:

  • Account:
    • Sign up and log in. Accounts information will be saved in a database.
    • Log in through Facebook.
    • Password changing feature.
    • Logout.
    • UI for “Login” and “Sign up” page.
  • Profile:
    • User’s avatar. (Upload an image)
    • User’s information: name, cooking style and cooking level, number of followers, number of recipes.
    • User’s posted posts from latest to oldest
    • UI for “Profile” page.
  • Posting:
    • Users can post after login.
    • The posting form will be specialized for posting a cooking recipe:
    • “Name” tab.
    • “Ingredient” tabs.
    • “Unit” tabs.
    • “Steps” tab.
    • “More details” tab.
    • Image/video upload feature for each tab.
    • Basic ingredients’ images in the database.
    • “Hashtag, highlight” feature to link to other related posts.
    • UI for “Posting”.
  • Interaction between users:
    • Wall feed: posts suggestions will be based on the user's activities and information.
    • “Follow” feature.
    • “Like & Dislike” feature.
    • “Report and hide” feature.
  • Extra features:
    • UI for “Saved recipes / My recipes” pages.
    • UI for “Wall feed”.
    • Search bar.
    • Notifications.
    • Convert saved posts into PDF files.

Any other features not included in this project are for further development which is also the limit of this project.
